# 🛠️ 智能維修助手 (Fault Tree Assistant)

這是一套設計給機台第一線操作/維修人員使用的動態網頁系統，能夠直接讀取並解析 `Draw.io` 匯出的流程圖 (XML 格式)，將複雜的維修步驟以引導式的「無腦問答卡片」呈現。

## 🌟 系統特色
*   **無需撰寫程式碼即可更新流程**：只需要繪製/修改 Draw.io 流程圖並覆蓋檔案即可。
*   **全自動零部署**：將 HTML/JS 搭配 `data.xml` 放在任何支援靜態網頁託管的空間（如 GitHub Pages）即可上線運作。
*   **免手動匯入**：使用者打開網頁會自動載入伺服器上的 `data.xml`，且提供關鍵字關聯搜尋及路徑麵包屑。
*   **離線備援**：支援斷網狀態下以流覽器快取 (Local Storage) 保留上次載入的流程資料，或接受本機手動上傳 XML。

## 🚀 如何部署至公開網路 (GitHub Pages)

如果您要把這套系統交接給下一手，並提供給所有人一條「永遠最新版」的網址，請按照以下步驟：

### 步驟 1：建立 GitHub Repository
1.  註冊/登入 [GitHub](https://github.com/).
2.  點擊右上角 `+` 選擇 **New repository**。
3.  命名您的專案（例如：`machine-maintain-assistant`），選擇 **Public**，點擊 **Create repository**。

### 步驟 2：上傳專案檔案
1.  進入剛剛建立的專案頁面，點擊 **"uploading an existing file"**。
2.  將您電腦上這個資料夾的以下四個檔案拖曳進去：
    *   `index.html`
    *   `style.css`
    *   `script.js`
    *   `維修流程.xml.xml` (請先將它改名為 **`data.xml`**)
3.  點擊綠色的 **Commit changes**。

### 步驟 3：開啟 GitHub Pages 免費網頁空間
1.  在專案頁面點擊最上方的 **Settings** 標籤。
2.  在左側邊欄找到 **Pages** 選單並點擊。
3.  在 **Build and deployment** 區塊，將 **Source** 下方的 `None` 改為 `main` (或 `master`) 分支，然後點擊 **Save**。
4.  等待約 1~2 分鐘，重新整理該頁面，您就會看到一行字寫著：
    `Your site is live at https://[您的帳號].github.io/machine-maintain-assistant/`
5.  **這條網址就是專案的正式入口了！把這條網址丟給維修群組即可！**

## 🔄 未來如何更新維修流程？

當公司購買了新的機台型號，或是原廠修改了某個除錯步驟時，負責維護這套系統的人員請按照以下步驟更新：

1.  開啟 [Draw.io](https://app.diagrams.net/) 修改流程圖，並存檔。
2.  點選檔案 ➔ 匯出為 ➔ **XML...**
3.  將下載下來的 XML 檔案重新命名為 **`data.xml`**。
4.  前往 GitHub 您的專案倉庫，點擊舊的 `data.xml` 檔案，選擇右上角的 **✏️ (Edit)** 或 **Delete** 然後重新上傳新的覆蓋掉。
5.  提交 (Commit changes)。過 1 分鐘後，第一線人員只要在手機上重新整理網頁，就會自動讀取新的流程了！

## ⚠️ 制圖規範 (Draw.io 格式要求)
為了能讓系統正確解析圖片，請確保您的製圖符合以下規則：
1.  **一定要標註分頁名稱**：在 Draw.io 的下方分頁標籤（Tab），一定要為每個機台命名（如：AI收瓶機），這會成為首頁最初始的選項。
2.  **上下左右對齊/連線**：
    *   建議使用箭頭實際將方塊連接起來 (`Source` ➔ `Target`)。
    *   如果**沒有使用箭頭**，系統會啟動智慧對齊：判斷「如果方塊 B 剛好在方塊 A 正下方貼齊，且兩者中心點 X 軸差距 < 30px」，就會自動將這兩個認為是相連的步驟。
3.  **如何呈現結果**：如果某個方塊*沒有進一步連接到下一個方塊*，系統就會自動將它視為「終點」（例如：要求使用者拍照回報）。
